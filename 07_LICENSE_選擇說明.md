# EOCMEET 授權選擇說明

本檔案不是正式法律意見，僅供 GitHub 發布前選擇授權方式參考。

## 情境一：只讓大家下載測試，不開放原始碼修改

可先使用「公開下載測試版」說明，不急著套用 MIT、Apache、GPL 等開源授權。

建議文字：

```text
本版本提供公開下載、測試、教育訓練與災害應變演練使用。
未經授權不得移除原作者資訊、不得宣稱為自行開發、不得轉售或用於違反法律之用途。
本系統仍屬 Release Candidate，正式勤務環境使用前，應由使用單位完成測試與資安審查。
```

## 情境二：要正式開源

如果你希望其他人可以自由使用、修改、再散布，建議選擇正式開源授權，例如：

- MIT License
- Apache License 2.0
- GPL-3.0 License

正式採用前，建議由單位或法律人員確認。

## GitHub 注意

Public repository 不等於正式開源。若要讓其他人明確擁有使用、修改、再散布的權利，通常需要放置明確的 LICENSE 檔案。
